# Introduction

Ce projet a été réalisé dans le cadre du **BUT 2 Informatique**, dans le module **Qualité de développement**, encadré par **M. L. Tran**.

---

## Contexte

Le thème du projet porte sur la conception et le développement d’un **portail web de gestion des paiements par cartes bancaires**, adossé à un entrepôt de données.

L’application permet la consultation, l’analyse et l’export d’informations financières pour différents profils utilisateurs :

  - commerçants
  - Product Owner (PO)
  - administrateurs

---

## Objectifs du projet

L’application développée, nommée **SafePay**, vise à fournir une interface centralisée permettant de :

- visualiser la trésorerie globale
- consulter les remises
- analyser les impayés
- générer des statistiques sous forme de graphiques
- exporter des rapports dans différents formats (PDF, CSV, XLS)
- gérer les droits d’accès selon les rôles utilisateur

---

## Méthodologie employée

Le projet a été mené selon une **méthodologie Agile (Scrum)**, avec une organisation basée sur des :

  - **User Stories (US)** pour définir les besoins fonctionnels
  - **Technical Stories (TS)** pour les aspects techniques

Les différentes fonctionnalités ont été regroupées sous forme **d’Epic Stories**, représentant les grands axes de développement :

  - gestion de la trésorerie
  - recherche et détail des remises
  - suivi des impayés
  - visualisation graphique des données
  - génération de rapports
  - authentification et gestion des rôles utilisateurs

---

## Technologies utilisées

  - **Back-end** : PHP  
  - **Front-end** : HTML, CSS, JavaScript  
  - **Base de données** : MySQL  
  - **Administration base de données** : phpMyAdmin  
  - **Documentation** : Zensical  
  - **Génération PDF** : FPDF  
  - **Versioning** : GitHub
